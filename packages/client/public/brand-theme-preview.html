Icon 锚点色

先固定图形里真正有语义的颜色，再从这些锚点延展成完整主题。这样后续 UI 不会长成另一套语言。

Brand 色阶

这组是主品牌色阶，适合按钮、选中态、品牌图形、强调块和关键状态。推荐 `500` 作为默认主题色，`600/700` 用于 hover 和 active。

生成逻辑：`50/100` 取 icon 高光系，`500` 取主品牌绿，`800` 取根系深色，之间做分段插值。

Sage 中性色阶

这组是偏绿灰的中性阶，不抢品牌主色，适合背景、边框、正文、分割线和静态模块。它决定的是“整体气质”，不是按钮颜色。

建议浅色界面默认落在 `50-200`，正文和标题主要使用 `700-950`。

语义 Token

主题最终要落到语义层，而不是页面里到处手写 `green-500`。这里给的是更适合产品 UI 的推荐分工。

组件效果

同一套色阶放进真实组件里，才能看出是否“稳”。这里重点看按钮层级、浅底信息卡、深色摘要卡和检索输入区的关系。
