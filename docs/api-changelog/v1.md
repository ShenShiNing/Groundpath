# API Changelog: v1

最后更新：2026-04-06

本文件记录 `v1` 时代 API 合同的历史变化。

- 当前接口现状以 `/api-docs` 暴露的 OpenAPI 文档为准
- 本文件只记录“变化”和“迁移影响”，不重复展开完整接口定义
- 2026-04-05 之前的条目为历史回填，依据 Git 提交和现有代码状态整理

## 2026-03-24

### Changed

- 统一 JSON 错误响应信封为 `success: false` + `error` 对象结构，并在可用时附带 `requestId`，验证失败细节通过 `error.details` 返回。

Compatibility: Conditionally compatible
Affected endpoints: 发生 JSON 错误响应的业务 API；该变更在 `/api/v1` 引入后成为其默认错误结构
Client action: Optional
Migration: 如果客户端依赖错误响应结构，统一按 `error.code`、`error.message`、`error.details?`、`error.requestId?` 解析，不再假设各中间件返回不同 shape。
Source: commit `a36a75c`

### Changed

- 收紧文档 AI 关键词和实体提取接口的请求契约，为两个端点分别定义显式请求 schema，并固定响应结构。

Compatibility: Conditionally compatible
Affected endpoints: `POST /api/v1/document-ai/{id}/analyze/keywords`, `POST /api/v1/document-ai/{id}/analyze/entities`
Client action: Optional
Migration: 关键词提取仅提交 `maxKeywords`、`language`；实体提取仅提交 `maxEntities`、`language`。客户端不要再依赖未声明字段被忽略或透传。
Source: commit `a8163f8`

## 2026-03-25

### Added

- 所有业务路由统一切换到 `/api/v1/**` 版本前缀，作为后续 API 演进的稳定基线。

Compatibility: Breaking
Affected endpoints: 业务 API 路由；不包括 `/api/files/**`、`/api/uploads/**`、`/api/hello`、`/health/*`
Client action: Required
Migration: 将业务调用从 `/api/...` 迁移到 `/api/v1/...`。文件访问与健康检查端点保持原路径，不要盲目追加 `/v1`。
Source: commit `22545c1`

### Changed

- 文档、垃圾桶和知识库列表接口改用游标分页；请求从 `page` 迁移为 `cursor`，响应分页元数据从 `page/totalPages` 迁移为 `hasMore/nextCursor`。

Compatibility: Breaking
Affected endpoints: `GET /api/v1/documents`, `GET /api/v1/documents/trash`, `GET /api/v1/knowledge-bases`, `GET /api/v1/knowledge-bases/{id}/documents`
Client action: Required
Migration: 移除基于 `page` / `totalPages` 的翻页逻辑，改为携带可选 `cursor` 请求参数，并基于 `pagination.hasMore` 与 `pagination.nextCursor` 继续翻页。
Source: commit `50b05a6`

## 2026-04-04

### Changed

- 用户资料相关路由从单数路径统一改为复数路径，和其余资源路由保持 REST 风格一致。

Compatibility: Breaking
Affected endpoints: `PATCH /api/v1/users/profile`, `PATCH /api/v1/users/email`, `POST /api/v1/users/avatar`
Client action: Required
Migration: 将旧路径 `/api/v1/user/profile`、`/api/v1/user/email`、`/api/v1/user/avatar` 更新为对应的 `/api/v1/users/...`。
Source: commit `9010f01`

## 2026-04-05

### Added

- 建立 `v1` API changelog 文档基线，并补回关键历史条目，后续所有对外可观察的 API 合同变更从这里持续追加。

Compatibility: Compatible
Affected endpoints: 全部 `/api/v1/**`，以及 `/api/files/**`、`/api/uploads/**`、`/api/hello`、`/health/*`
Client action: None
Migration: None
Source: docs/api-changelog bootstrap

## 2026-04-06

### Changed

- 聊天会话创建接口在传入 `knowledgeBaseId` 时，改为与会话更新接口一致，立即校验知识库存在且归属当前用户。

Compatibility: Conditionally compatible
Affected endpoints: `POST /api/v1/chat/conversations`
Client action: Optional
Migration: 客户端只能提交当前用户拥有的 `knowledgeBaseId`。此前可能被接受的无效或非本人知识库 ID，现在会返回 `404 KNOWLEDGE_BASE_NOT_FOUND`。
Source: commit `ff8c813`
